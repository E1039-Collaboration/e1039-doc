1 # A Program to Run Full Chain of E1039 Simulation
2 
3 ## Environment
4 
5 This program depends on the E1039 resource+share+core packages.
6 Unless you have a special reason, you are recommended to use "spinquestgpvm01.fnal.gov",
7 where the E1039 packages are ready to use.
8 
9 You can just execute the following command to set up the environment properly;
10 ```
11 source /e906/app/software/osg/software/e1039/this-e1039.sh
12 ```
13 You have to do so when you start a new shell environment (i.e. text terminal).
14 
15 ## Download
16 
17 You move to a working directory and check out the repository;
18 ```
19 cd /path/to/your_working_directory
20 git clone https://github.com/E1039-Collaboration/e1039-analysis.git
21 ```
22 If you are a member of the GitHub E1039 group, you better use the following command to obtain the write access;
23 ```
24 git clone git@github.com:E1039-Collaboration/e1039-analysis.git
25 ```
26 
27 ## Test Process
28 
29 You run the following commands to run a small test.
30 It will take 10-20(?) minutes.
31 Note that you have to set up the environment beforehand, as explained above.
32 
33 ```
34 cd e1039-analysis/SimChainDev
35 root -b -q Fun4Sim.C
36 ```
37 
38 By default a simple muon-pair event (via PHG4SimpleEventGenerator) is simulated.
39 You can/should change various options to simulate what you need.
40 
41 The simulation result is written into multiple ROOT files.
42 The main output file is "DST.root", which is structured in the E1039 standard data format.
43 It is usually analyzed by `e1039-analysis/AnaSimDst`.
44 
45 ## Large Process
46 
47 When you want a large set of simulated events,
48 you should run the simulation on the grid.
49 The procedure is explained in
50 [this Wiki page](https://github.com/E1039-Collaboration/e1039-wiki/wiki/Submit-jobs-to-the-grid).
